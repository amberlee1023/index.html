Hosted Timesheet App

Starter plan for employee logins and manager approvals.
Recommended build

Move this into a real hosted web app with logins, saved submissions, and manager approvals.

For your team, the cleanest setup is a hosted app where employees sign in, submit weekly time by client and project, and managers review entries in one shared dashboard.

Hosting
Vercel or Netlify

Fast deployment for the web front end.

Login
Supabase Auth

Employee and manager sign-in with role control.

Database
Supabase Postgres

Central timesheet records, clients, projects, and approvals.

What employees do
  • Open one live web link in any browser.
  • Sign in with their company email and password or a one-time login link.
  • Select the week, choose client and project dropdowns, enter daily hours, and submit.
  • View their own past timesheets and submission status.
What managers do
  • Sign in through the same app URL.
  • See all employee submissions in one dashboard.
  • Approve, reject, or export timesheets by week, employee, client, or project.
  • Manage the client list, project list, and employee access.
Database tables
users (id, name, email, role) clients (id, name) projects (id, client_id, name) timesheets (id, user_id, week_of, approver_id, status, submitted_at) time_entries (id, timesheet_id, client_id, project_id, mon, tue, wed, thu, fri, sat, sun, total_hours)
